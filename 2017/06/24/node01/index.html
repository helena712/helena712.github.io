생활코딩 https://opentutorials.org/course/2136/11949

웹 애플리케이션은 사용자의 입력주소에 따라 페이지를 보여주고, 패스에 따라 여러 아웃풋을 보여준다.

쿼리 스트링

URL 구성

  • http(웹페이지를 요청하고 받아내기 위한 프로토콜)+도메인(서버컴퓨터 위치 주소)+패스+ ?(쿼리스트링)

쿼리스트링은 path로 받는 주소 외에 추가적으로 넘겨받는 값이 있을 때 사용한다.

API Reference(명령어 사전) 참고하기

사용자가 user 로 들어왔을 때, express는 function을 호출하고,
이 function는 첫번째 인자로 req를 두번째 인자로 res를 전달한다.

localhost:3000/topic?id=1

req객제가 가지는 query라는 객체에 id라는 프로퍼티를 이용해 값을 이용할 수 있다.
어떤 쿼리 스트링을 전달하는지에 따라 웹어플리케이션이 다르게 동작한다.

시멘틱URL

쿼리스트링보다 의미론적으로 url로 만드는 방식
localhost:3000/topic/id

id라는 가변적인 정보를 잡아낸다.
이 가변적 정보는 params라는 객체를 사용해 이용할 수 있다.

서버와 웹브라우저가 상호작용하는 방식

  • GET
    웹브라우저에서 어떤 정보를 서버에 요청

localhost:3000/form_receiver’?title=hello&description=world

form은 action을 통해 정보를 어디로 전달할지 알려주고,
웹브라우저는 form에 따라 정보를 가진 url를 생성해 서버로 전달해 준다.
** method를 생략하면 GET이 됨

  • POST
    사용자의 정보를 서버로 전달

method를 POST으로 하면, 데이터를 서버로 전송하지만,
url로 전송하지는 않는다. 그렇기에 url주소는 같다.

기본적으로 post로 전달된 것을 undefind여서,
body-parser라는 미들웨어(확장기능)을 사용해야한다.

1. body-parser 설치
npm install body-parser —saver
2. body-parser 모듈 가져오기
var bodyParser = require(‘body=-parser’);
3. body-parser 모듈 사용하기
app.use(bodyParser.urlencoded({ extended: false}))
** use는 애플리케이션에 붙인다는 의미

애플리케이션에 들어오는 모든 요청들은 미들웨어를 통과한 후 사용할수 있다.

이제 body-parser가 실행이 될 때,
post방식이 있다면, body라는 객체를 사용할 수 있다.
